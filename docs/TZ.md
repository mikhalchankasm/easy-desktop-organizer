# Техническое задание

## Программа-органайзер рабочего стола Windows, аналог iTop Easy Desktop

## 1. Назначение программы

Разработать Windows-приложение для организации рабочего стола пользователя с помощью визуальных контейнеров, автоматической сортировки файлов/ярлыков, быстрого поиска, сохранения раскладок и скрытия элементов рабочего стола.

Программа должна работать локально на компьютере пользователя и не требовать подключения к интернету для основной функциональности.

## 2. Целевая платформа

Приложение разрабатывается только для Windows.

Поддерживаемые ОС:

* Windows 10 x64
* Windows 11 x64

Минимальные требования:

* .NET Desktop Runtime или self-contained сборка
* 4 ГБ RAM
* разрешение экрана от 1366×768
* поддержка одного или нескольких мониторов

## 3. Основная идея

Программа должна создавать поверх рабочего стола управляемые визуальные зоны — "коробки", в которые пользователь может помещать:

* ярлыки программ;
* файлы;
* папки;
* ссылки;
* временные рабочие материалы.

Коробки должны вести себя как элементы рабочего пространства: их можно перемещать, изменять по размеру, переименовывать, настраивать, скрывать и сохранять в раскладках.

## 4. Основные сценарии использования

### 4.1. Организация рабочего стола

Пользователь запускает программу, нажимает кнопку "Автоорганизация", после чего приложение анализирует элементы рабочего стола и раскладывает их по категориям:

* Документы;
* Изображения;
* Видео;
* Архивы;
* Папки;
* Программы;
* Ссылки;
* Прочее.

### 4.2. Ручное создание коробок

Пользователь может вручную создать коробку, задать ей имя и перетащить туда нужные элементы.

Пример:

* "Проект 240101"
* "AutoCAD"
* "Временные файлы"
* "Скриншоты"
* "Документы на проверку"

### 4.3. Быстрое скрытие рабочего стола

Пользователь дважды кликает по свободной области рабочего стола или нажимает горячую клавишу. Все коробки и элементы временно скрываются.

Повторное действие возвращает рабочее пространство.

### 4.4. Сохранение раскладок

Пользователь может сохранить текущую раскладку:

* положение коробок;
* размеры;
* названия;
* содержимое;
* настройки внешнего вида;
* привязку к монитору.

Можно создать несколько раскладок:

* "Работа";
* "Дом";
* "Проект 1";
* "Демонстрация экрана";
* "Чистый рабочий стол".

### 4.5. Быстрый поиск

Пользователь нажимает горячую клавишу и ищет файл, папку или ярлык по имени. Программа показывает результат и позволяет:

* открыть файл;
* открыть папку расположения;
* показать элемент в коробке;
* переместить элемент в другую коробку.

## 5. Функциональные требования

## 5.1. Модуль коробок

Программа должна поддерживать создание визуальных контейнеров на рабочем столе.

Каждая коробка должна иметь:

* название;
* позицию на экране;
* ширину и высоту;
* цвет фона;
* прозрачность;
* цвет рамки;
* режим отображения элементов;
* внутреннюю прокрутку;
* контекстное меню.

Обязательные действия:

* создать коробку;
* удалить коробку;
* переименовать коробку;
* изменить размер;
* переместить;
* закрепить позицию;
* скрыть;
* свернуть;
* развернуть;
* очистить содержимое;
* открыть настройки коробки.

## 5.2. Содержимое коробок

В коробки можно помещать:

* `.lnk`;
* `.url`;
* файлы любых расширений;
* папки;
* сетевые пути;
* UNC-ссылки вида `\\server\share\folder`.

Для каждого элемента необходимо хранить:

* имя;
* полный путь;
* тип;
* расширение;
* иконку;
* дату добавления;
* дату изменения файла;
* ID коробки;
* порядок отображения внутри коробки.

Действия с элементами:

* открыть;
* открыть расположение;
* переименовать;
* удалить с рабочего стола;
* убрать из коробки без удаления файла;
* переместить в другую коробку;
* скопировать путь;
* показать свойства файла.

## 5.3. Перетаскивание

Программа должна поддерживать drag-and-drop:

* с рабочего стола в коробку;
* из Проводника Windows в коробку;
* из коробки в другую коробку;
* из коробки обратно на рабочий стол;
* изменение порядка элементов внутри коробки.

При переносе файла программа не должна физически перемещать файл без явного разрешения пользователя.

По умолчанию коробка хранит ссылку на объект, а не переносит сам файл.

## 5.4. Автоматическая сортировка

Должен быть режим автоматической сортировки элементов рабочего стола.

Категории по умолчанию:

| Категория   | Расширения                                               |
| ----------- | -------------------------------------------------------- |
| Документы   | `.doc`, `.docx`, `.xls`, `.xlsx`, `.pdf`, `.txt`, `.rtf` |
| Изображения | `.png`, `.jpg`, `.jpeg`, `.bmp`, `.gif`, `.webp`         |
| Чертежи     | `.dwg`, `.dxf`, `.rvt`, `.ifc`, `.nwd`, `.nwf`           |
| Архивы      | `.zip`, `.rar`, `.7z`                                    |
| Видео       | `.mp4`, `.avi`, `.mov`, `.mkv`                           |
| Скрипты     | `.py`, `.ps1`, `.bat`, `.cmd`, `.js`, `.cs`              |
| Ярлыки      | `.lnk`, `.url`                                           |
| Папки       | директории                                               |
| Прочее      | остальные файлы                                          |

Пользователь должен иметь возможность редактировать правила сортировки.

Правило сортировки должно включать:

* имя правила;
* тип условия;
* список расширений;
* ключевые слова в имени;
* путь источника;
* целевую коробку;
* приоритет;
* статус включено/выключено.

## 5.5. Автоматическое отслеживание рабочего стола

Программа должна отслеживать изменения в папках:

* `Desktop` текущего пользователя;
* `Public Desktop`;
* дополнительные папки, добавленные пользователем.

При появлении нового файла программа должна:

1. определить категорию;
2. применить пользовательские правила;
3. предложить переместить в подходящую коробку или сделать это автоматически, если включен соответствующий режим.

## 5.6. Режим "чистый рабочий стол"

Программа должна иметь команду:

* скрыть все коробки;
* показать все коробки;
* скрыть только содержимое;
* оставить видимыми только закрепленные коробки.

Управление:

* двойной клик по рабочему столу;
* горячая клавиша;
* пункт меню в трее.

Горячая клавиша по умолчанию:

```text
Ctrl + Alt + D
```

## 5.7. Быстрый поиск

Поиск должен работать по:

* названию файла;
* расширению;
* имени коробки;
* пути;
* типу элемента.

Окно поиска должно открываться горячей клавишей:

```text
Ctrl + Space
```

Функции поиска:

* мгновенный вывод результатов;
* открытие найденного элемента;
* открытие папки расположения;
* фильтр по типу;
* фильтр по коробке;
* история последних поисков.

## 5.8. Раскладки рабочего пространства

Программа должна позволять создавать, сохранять и переключать раскладки.

Раскладка включает:

* список коробок;
* позиции коробок;
* размеры;
* монитор;
* внешний вид;
* состав элементов;
* правила сортировки;
* настройки скрытия.

Функции:

* создать раскладку;
* сохранить текущую;
* дублировать;
* переименовать;
* удалить;
* экспортировать в файл;
* импортировать из файла;
* восстановить раскладку после перезапуска Windows.

## 5.9. Поддержка нескольких мониторов

Программа должна корректно работать с несколькими мониторами.

Требования:

* коробки должны запоминать монитор;
* при отключении монитора коробки должны переноситься на активный экран;
* после повторного подключения монитора раскладка должна восстанавливаться;
* программа должна учитывать разные DPI и масштабирование Windows;
* не допускать появления коробок за пределами видимой области.

## 5.10. Автозапуск

Программа должна иметь настройку автозапуска вместе с Windows.

Варианты:

* через Startup folder;
* через ключ реестра `HKCU\Software\Microsoft\Windows\CurrentVersion\Run`;
* через планировщик задач Windows.

Рекомендуемый вариант для MVP:

```text
HKCU\Software\Microsoft\Windows\CurrentVersion\Run
```

## 5.11. Иконка в системном трее

Программа должна работать в фоне и иметь иконку в системном трее.

Меню трея:

* показать/скрыть коробки;
* открыть поиск;
* открыть настройки;
* переключить раскладку;
* включить/отключить автоорганизацию;
* выйти из программы.

## 5.12. Настройки внешнего вида

Для каждой коробки:

* цвет фона;
* прозрачность;
* цвет заголовка;
* цвет текста;
* размер шрифта;
* плотность элементов;
* размер иконок;
* режим сетки;
* режим списка;
* скругление углов;
* тень;
* рамка.

Глобальные настройки:

* тема светлая/темная/системная;
* язык интерфейса;
* масштаб интерфейса;
* анимации включены/выключены.

## 5.13. Виджеты

В MVP виджеты не обязательны.

Для расширенной версии можно добавить:

* часы;
* календарь;
* заметки;
* список задач;
* быстрые ссылки;
* последние файлы;
* скриншоты дня.

Виджеты должны быть реализованы как отдельные панели, не смешанные с коробками файлов.

## 5.14. Приватная коробка

Для расширенной версии можно реализовать защищенную коробку.

Функции:

* пароль на открытие;
* блокировка после тайм-аута;
* скрытие содержимого;
* шифрование локальной базы;
* запрет отображения файлов в поиске без разблокировки.

Важно: если реализуется приватная коробка, необходимо явно определить, что именно защищается:

* только отображение в программе;
* или сами файлы физически шифруются на диске.

Для MVP рекомендуется не реализовывать физическое шифрование файлов.

## 6. Нефункциональные требования

## 6.1. Производительность

Программа не должна заметно замедлять работу Windows.

Целевые показатели:

* запуск приложения: до 3 секунд;
* открытие поиска: до 300 мс;
* реакция UI: до 100 мс;
* потребление RAM в фоне: желательно до 150 МБ;
* CPU в простое: менее 1%;
* корректная работа с 1000+ элементами рабочего стола.

## 6.2. Надежность

Программа должна:

* не удалять файлы без подтверждения;
* не перемещать файлы без подтверждения;
* сохранять резервную копию настроек;
* восстанавливать последнее состояние после сбоя;
* вести лог ошибок;
* не блокировать запуск Windows;
* не ломать стандартный рабочий стол Windows.

## 6.3. Безопасность

Все данные должны храниться локально.

Запрещено:

* отправлять список файлов на внешний сервер;
* собирать телеметрию без явного согласия;
* изменять системные файлы Windows;
* перехватывать ввод пользователя за пределами необходимых горячих клавиш.

## 6.4. Совместимость

Программа должна корректно работать с:

* стандартным рабочим столом Windows;
* Проводником Windows;
* ярлыками `.lnk`;
* сетевыми путями;
* OneDrive Desktop;
* корпоративными профилями пользователей;
* разными DPI;
* несколькими мониторами.

## 7. Архитектура приложения

## 7.1. Рекомендуемый стек

Рекомендуемый стек для разработки:

* язык: C#;
* платформа: .NET Desktop;
* UI: WPF или WinUI 3;
* база данных: SQLite;
* хранение настроек: JSON + SQLite;
* установщик: MSIX или Inno Setup;
* логирование: Serilog или встроенный логгер;
* файловый мониторинг: `FileSystemWatcher`.

Для MVP предпочтительно:

```text
C# + WPF + SQLite + Inno Setup
```

Причина: WPF проще для кастомных overlay-окон, drag-and-drop, трея и быстрого прототипирования.

## 7.2. Основные модули

Приложение должно быть разделено на модули:

1. `DesktopOverlay` — отображение коробок, перемещение, изменение размера, прорисовка UI.
2. `BoxManager` — создание и удаление коробок, хранение структуры, управление элементами.
3. `FileIndexService` — индексация рабочего стола, отслеживание новых файлов, обновление данных.
4. `AutoSortService` — применение правил, распределение элементов по коробкам.
5. `LayoutService` — сохранение и восстановление раскладок.
6. `SearchService` — быстрый поиск по индексу.
7. `SettingsService` — глобальные настройки, настройки внешнего вида, горячие клавиши.
8. `TrayService` — меню в системном трее.
9. `StartupService` — управление автозапуском.
10. `LoggingService` — логирование ошибок и действий.

## 8. Модель данных

## 8.1. Таблица `Boxes`

Поля: `Id`, `Name`, `LayoutId`, `MonitorId`, `X`, `Y`, `Width`, `Height`, `BackgroundColor`, `Opacity`, `IsLocked`, `IsCollapsed`, `IsHidden`, `SortOrder`, `CreatedAt`, `UpdatedAt`.

## 8.2. Таблица `Items`

Поля: `Id`, `BoxId`, `Name`, `FullPath`, `ItemType`, `Extension`, `IconCacheKey`, `DisplayOrder`, `IsMissing`, `CreatedAt`, `UpdatedAt`.

## 8.3. Таблица `Layouts`

Поля: `Id`, `Name`, `IsDefault`, `CreatedAt`, `UpdatedAt`.

## 8.4. Таблица `SortRules`

Поля: `Id`, `Name`, `ConditionType`, `Extensions`, `Keywords`, `SourcePath`, `TargetBoxId`, `Priority`, `IsEnabled`.

## 8.5. Таблица `Settings`

Поля: `Key`, `Value`, `UpdatedAt`.

## 9. UI-требования

## 9.1. Главное поведение

После запуска программа не должна открывать большое главное окно.

Основное взаимодействие происходит через:

* коробки на рабочем столе;
* контекстные меню;
* системный трей;
* окно настроек;
* окно поиска.

## 9.2. Коробка

Коробка должна содержать:

* заголовок;
* кнопку свернуть;
* кнопку настроек;
* область элементов;
* полосу прокрутки при необходимости;
* возможность изменения размера за края.

Контекстное меню коробки:

* переименовать;
* изменить цвет;
* закрепить;
* свернуть;
* скрыть;
* сортировать содержимое;
* открыть настройки;
* удалить коробку.

## 9.3. Элемент внутри коробки

Элемент отображается как иконка + имя, либо строка списка.

Контекстное меню элемента:

* открыть;
* открыть расположение;
* переименовать;
* удалить файл;
* убрать из коробки;
* переместить в коробку;
* копировать путь;
* свойства.

## 9.4. Окно настроек

Разделы: Общие, Коробки, Автоорганизация, Раскладки, Горячие клавиши, Внешний вид, Мониторы, Автозапуск, Резервное копирование, О программе.

## 10. Горячие клавиши

По умолчанию:

| Действие                | Комбинация       |
| ----------------------- | ---------------- |
| Показать/скрыть коробки | `Ctrl + Alt + D` |
| Открыть поиск           | `Ctrl + Space`   |
| Создать коробку         | `Ctrl + Alt + N` |
| Сохранить раскладку     | `Ctrl + Alt + S` |
| Переключить раскладку   | `Ctrl + Alt + L` |

Пользователь должен иметь возможность изменить горячие клавиши.

## 11. Логирование

Программа должна вести лог: запуск приложения; ошибки чтения/записи; ошибки доступа к файлам; ошибки восстановления раскладки; сбои FileSystemWatcher; исключения UI; ошибки базы данных.

Логи хранить в `%LOCALAPPDATA%\DesktopOrganizer\Logs`.

## 12. Хранение данных

Основная папка приложения: `%LOCALAPPDATA%\DesktopOrganizer`.

```text
DesktopOrganizer
├─ app.db
├─ settings.json
├─ icon-cache
├─ backups
└─ logs
```

Создавать резервную копию: перед обновлением; перед массовой автоорганизацией; вручную по команде пользователя; один раз в день при активном использовании.

## 13. Установщик

Требования:

* установка в `%LOCALAPPDATA%` или `%ProgramFiles%`;
* создание ярлыка в меню Пуск;
* опциональное добавление в автозагрузку;
* корректное удаление;
* сохранение пользовательских настроек при обновлении;
* возможность полного удаления с настройками;
* silent install: `DesktopOrganizerSetup.exe /silent`.

## 14. MVP-версия

В MVP обязательно реализовать:

1. запуск Windows-приложения;
2. иконку в трее;
3. создание коробок;
4. перемещение и изменение размера коробок;
5. добавление файлов/папок/ярлыков в коробки;
6. drag-and-drop;
7. сохранение состояния;
8. восстановление после перезапуска;
9. автосортировку по расширениям;
10. скрытие/показ коробок горячей клавишей;
11. поиск по элементам;
12. поддержку Windows 10/11;
13. поддержку нескольких мониторов на базовом уровне;
14. локальное хранение данных в SQLite;
15. установщик.

## 15. Не входит в MVP

Облачная синхронизация; аккаунты пользователей; AI-ассистент; магазин обоев; физическое шифрование файлов; сложные виджеты; синхронизация между компьютерами; мобильное приложение; серверная часть.

## 16. Расширенная версия

Приватная коробка; заметки; календарь; задачи; скриншоты рабочего дня; шаблоны раскладок; импорт/экспорт настроек; профили для разных проектов; правила автоорганизации по регулярным выражениям; корпоративные политики; silent deploy; централизованный конфиг; интеграция с OneDrive/Google Drive/сетевыми папками; журнал активности пользователя по рабочим файлам.

## 17. Особые требования для инженерного применения

Преднастроенные категории: AutoCAD; Navisworks; AVEVA E3D; Revit; Tekla; Excel; PDF; Скриншоты; Проектная документация; Временные файлы; Проверка; Замечания; Архив.

## 18. Критерии приемки

1. устанавливается на Windows 10 и Windows 11;
2. запускается без прав администратора;
3. создает минимум 5 коробок;
4. сохраняет их положение после перезапуска;
5. позволяет добавлять файлы drag-and-drop;
6. открывает файлы из коробок;
7. корректно работает с ярлыками `.lnk`;
8. выполняет автоорганизацию по расширениям;
9. скрывает и показывает коробки горячей клавишей;
10. выполняет поиск по добавленным элементам;
11. не удаляет и не перемещает файлы без подтверждения;
12. работает при подключении второго монитора;
13. не потребляет более 150 МБ RAM в обычном режиме;
14. имеет лог ошибок;
15. имеет установщик.

## 19. Риски

| Риск                              | Решение                                               |
| --------------------------------- | ----------------------------------------------------- |
| Конфликт с рабочим столом Windows | Не заменять Explorer, а использовать overlay-окна     |
| Проблемы с DPI                    | Использовать DPI-aware режим                          |
| Потеря путей к файлам             | Помечать элемент как отсутствующий                    |
| Случайное удаление файла          | Всегда запрашивать подтверждение                      |
| Много элементов                   | Использовать индекс и виртуализацию списка            |
| OneDrive Desktop                  | Не вмешиваться в синхронизацию, хранить только ссылки |

## 20. Рекомендованный план разработки

1. **Прототип**: WPF-приложение; окно-коробка поверх рабочего стола; перемещение; изменение размера; сохранение координат.
2. **Работа с элементами**: drag-and-drop; добавление файлов; открытие файлов; контекстное меню; SQLite-хранение.
3. **Автоорганизация**: сканирование Desktop; правила по расширениям; создание коробок по категориям; безопасное распределение элементов.
4. **Поиск и трей**: системный трей; горячие клавиши; окно поиска; скрытие/показ коробок.
5. **Раскладки и мониторы**: сохранение раскладок; переключение раскладок; поддержка нескольких мониторов; DPI-проверка.
6. **Установщик и тестирование**: Inno Setup; автозагрузка; логирование; тесты на Windows 10/11; проверка обновления версии.

## 21. Итоговая формулировка задачи для разработчика

Необходимо разработать локальное Windows-приложение для организации рабочего стола с помощью настраиваемых визуальных коробок. Программа должна позволять группировать файлы, папки и ярлыки, выполнять автоорганизацию по правилам, сохранять раскладки, быстро искать элементы и скрывать рабочее пространство по горячей клавише. Приложение должно работать без сервера, хранить данные локально, не изменять физическое расположение файлов без подтверждения пользователя и быть совместимым с Windows 10/11.
