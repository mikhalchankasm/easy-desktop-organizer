# Easy Desktop Organizer

Лёгкий локальный органайзер рабочего стола для Windows 10/11. Раскладывает файлы, папки и ярлыки по визуальным «коробкам» поверх рабочего стола — вдохновлено iTop Easy Desktop и Stardock Fences. Работает без интернета и без серверов, все данные хранятся локально, файлы физически не перемещаются.

> A lightweight, local desktop organizer for Windows. Group your desktop files, folders and shortcuts into customizable boxes. No cloud, no account, no telemetry — everything stays on your machine.

## Возможности

- **Коробки** поверх рабочего стола: перемещение, изменение размера, сворачивание, закрепление, переименование.
- **Внешний вид коробки** (⚙ на заголовке): 12 цветов, прозрачность, размер значков (мелкие / обычные / крупные).
- **Drag-and-drop**: из Проводника или с рабочего стола в коробку, между коробками, ручная сортировка внутри коробки.
- **«Перемещение» без переноса файла**: ярлык исчезает со стола (скрывается атрибутом), но физически остаётся на месте — путь не меняется, OneDrive не затрагивается. При выходе из программы всё возвращается на стол.
- **Автоорганизация**: раскладывает рабочий стол по категориям (документы, изображения, чертежи `.dwg/.rvt/.nwd/.ifc`, архивы, видео, скрипты, ярлыки, …) — с подтверждением перед массовой операцией.
- **Магнитное выравнивание** коробок с направляющими линиями — при перемещении и при изменении размера.
- **Быстрый поиск** по именам, путям, расширениям и именам коробок (`Ctrl+Space`).
- **Режим «чистый рабочий стол»**: скрыть/показать все коробки горячей клавишей, двойным кликом по пустому столу или из трея.
- **Системный трей**, **автозапуск с Windows**, **журнал ошибок**.
- Несколько мониторов и Per-Monitor V2 DPI; запуск без прав администратора.

## Стек

- C# / .NET 8, WPF (+ WinForms только для иконки в трее)
- SQLite (`Microsoft.Data.Sqlite`)
- Inno Setup для установщика

## Сборка и запуск

```powershell
dotnet build src/DesktopOrganizer/DesktopOrganizer.csproj
dotnet run --project src/DesktopOrganizer/DesktopOrganizer.csproj
```

Требуется [.NET 8 SDK](https://dotnet.microsoft.com/download) на Windows 10/11 (x64). Главного окна нет — после запуска появляется значок в трее и коробки на рабочем столе.

## Тесты

```powershell
dotnet test tests/DesktopOrganizer.Tests/DesktopOrganizer.Tests.csproj
```

Тесты ссылаются на отдельную сборку **`DesktopOrganizer.Core`** (`net8.0`, без WPF/Win32), поэтому `dotnet test` не пересобирает `DesktopOrganizer.exe` и проходит даже при запущенном приложении.

Покрывают чистую логику восстановления:

- **`RestorePlanner`** (объединение журнала и БД): fallback из БД при пустом журнале, дедуп одного файла из нескольких коробок, отбрасывание битов вне `Hidden|System`, приоритет журнала над БД, проброс `AddedAttributes=0`.
- **`RecoveryPlan`** (политика результата): флаги БД чистятся только для восстановленных (`Restored`/`FileGone`), `-1` при неполном чтении журнала или БД (деинсталлятор не сотрёт данные), восстановленное по журналу помечается даже при сбое чтения БД.

## Установщик

```powershell
dotnet publish src/DesktopOrganizer/DesktopOrganizer.csproj -c Release -r win-x64 --self-contained -p:PublishSingleFile=true
iscc installer/DesktopOrganizer.iss   # требуется Inno Setup 6
```

Результат: `installer/Output/DesktopOrganizerSetup.exe` (тихая установка: `/VERYSILENT /NORESTART`).

## Управление

| Действие | Как |
| --- | --- |
| Показать/скрыть все коробки | `Ctrl+Alt+D`, двойной клик по пустому рабочему столу, двойной клик по значку в трее |
| Быстрый поиск | `Ctrl+Space` |
| Новая коробка | `Ctrl+Alt+N`, контекстное меню коробки, меню трея |
| Автоорганизация рабочего стола | меню трея |
| Настройки коробки (цвет, прозрачность, значки) | кнопка ⚙ на коробке или правый клик |
| Переместить коробку | перетаскивание за заголовок (прилипает к соседям) |
| Свернуть коробку | кнопка «—» или двойной клик по заголовку |
| Изменить размер | за края окна (прилипает к соседям) |
| Добавить элементы | перетащить файлы из Проводника или с рабочего стола в коробку |
| Порядок элементов | перетаскивание значка внутри коробки |

## Безопасность данных

- Коробки хранят **ссылки** на файлы; файлы не перемещаются и не удаляются без явного подтверждения.
- «Перемещение» ярлыка в коробку = скрытие его иконки со стола (атрибут Hidden+System). «Вернуть на рабочий стол», очистка или удаление коробки возвращают ярлык. **При выходе из приложения все скрытые ярлыки возвращаются** — без запущенной программы доступ к файлам не теряется.
- Удаление файла с диска — только через подтверждение и в Корзину.
- `Win+D` / «Показать рабочий стол» коробки не сворачивает (они часть слоя рабочего стола).

```
%LOCALAPPDATA%\DesktopOrganizer\
├─ app.db      — SQLite: коробки, элементы, настройки
└─ Logs\       — журналы app-YYYYMMDD.log
```

> **Обновляющимся с ранних сборок:** для элементов, скрытых до появления точного учёта атрибутов, при первом запуске предполагается, что приложение добавило `Hidden+System`. Если такой файл *до* добавления в коробку уже был «скрытым» вручную, после возврата на стол он перестанет быть скрытым. Новые добавления хранят точный набор изменённых битов и этой особенности не подвержены.

## Структура проекта

```
src/DesktopOrganizer/      — приложение (WPF)
  Data/                    — слой SQLite
  Models/                  — модели данных
  Services/                — иконки, горячие клавиши, автозапуск, автосортировка, хуки рабочего стола
  UI/                      — окна коробки, поиска, настроек, оверлей выравнивания
src/DesktopOrganizer.Core/ — чистая логика восстановления (RestorePlanner, RecoveryPlan, HideAttributes), без Win32/WPF
tests/DesktopOrganizer.Tests/ — xUnit поверх Core
installer/                 — скрипт Inno Setup
tools/                  — генератор иконки приложения
docs/TZ.md              — техническое задание (RU)
```
